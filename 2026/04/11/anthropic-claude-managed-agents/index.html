Anthropic 发布 Claude Managed Agents：把 Agent 基础设施变成云服务

2026 年初，Anthropic 和 OpenAI 几乎同时发布了关于 Harness Engineering 的技术文章，提出了一个简洁公式：Agent = Model + Harness。模型提供智能，Harness 负责让智能落地生产。

现在 Anthropic 把这个公式变成了产品——Claude Managed Agents，一套用于构建和部署云托管 Agent 的 API 套件，已进入公开 Beta。

解决了什么问题？

构建生产级 Agent 远不止调用几次 API。你需要：

  • 沙箱化执行环境：代码在隔离容器里跑
  • 检查点机制：长任务中断后能恢复
  • 凭证管理：OAuth token 不能暴露给 Agent
  • 权限控制：Agent 不能随意访问敏感系统
  • 执行追踪：出了问题要能回溯

这些基础设施工作可能耗费数月，而用户看不到任何功能进展。Claude Managed Agents 把这些复杂性全部接管。开发者只需定义 Agent 的任务、工具和护栏，Anthropic 负责运行。

核心架构：脑手分离

Anthropic 官方工程博客详细解释了架构设计思路。他们一开始把所有组件塞进一个容器——Claude 模型（大脑）、沙箱环境（手脚）、会话记忆都在一起。

问题是：这等于养了一只宠物（Pet）。容器挂了，会话就丢了。容器卡住了，得人工去修。调试需要进入容器，但容器里有用户数据，安全隐患。

解决方案是解耦——把”大脑”从”手脚”分开：

组件 职责 接口
Brain（大脑） Claude + Harness 编排循环 无状态，可随时重启
Session（会话） 持久化事件日志 getEvents() / emitEvent()
Sandbox（沙箱） 代码执行环境 execute(name, input) → string

每个组件都是”牛群”（Cattle）而不是”宠物”（Pet）：

  • 容器挂了？ Harness 捕获错误，启动新容器
  • Harness 崩了？ 新 Harness 用 wake(sessionId) 恢复，从日志最后一条事件继续
  • 凭证安全？ Token 存在 Vault 里，Sandbox 永远碰不到

这个设计带来了显著的性能提升：解耦后 **p50 TTFT（首 Token 响应时间）下降约 60%，p95 下降超过 90%**。

三个关键设计模式

Anthropic 在设计 Managed Agents 时总结了三个从 Claude Code 实践中得到的模式：

模式一：用 Claude 已知的工具

Claude 在 SWE-bench Verified 上从 49% 提升到顶级水平，始终只用两个工具：bash + 文本编辑器。不是为每个任务设计专用工具，而是让 Claude 用通用工具自己组合出解决方案。

模式二：让 Claude 自主决策

传统模式下，每个工具调用的结果都要过 Claude 的上下文窗口，既慢又贵。给 Claude 一个代码执行工具（bash），让它自己编排工具调用链——只有最终结果才进入上下文。

实测效果：在 BrowseComp 基准测试中，给 Opus 4.6 这个能力后，准确率从 **45.3% 提升到 61.6%**。

另一个点是 Skills 渐进式展开：系统提示词里只放简短描述（YAML 前言），需要时 Claude 才读取完整 skill 内容，避免注意力浪费。

模式三：谨慎设置边界

不是所有操作都该走 bash。难以逆转的操作（外部 API 调用）应该提升为专用工具，这样 Harness 可以拦截、审计、要求用户确认。

四大核心能力

能力 说明
生产级 Agent 安全沙箱、身份验证、工具执行全部平台托管
长运行会话 Agent 可自主工作数小时，进度持久化，断连不丢失
多 Agent 协调 主 Agent 可派生子 Agent 并行处理，汇总结果
可信治理 作用域权限、身份管理、执行追踪内置

定价方面，除标准 token 费用外，按会话活跃时间收费：**$0.08/小时**。

谁在用？

  • Notion：Claude 直接集成到工作空间，团队可委托编码、文档、演示文稿
  • Sentry：调试 Agent 自动编写补丁并打开 PR，数周内完成集成
  • Asana：AI Teammates 在项目中与人类并肩工作
  • Rakuten：跨部门企业 Agent，每个一周内部署完成
  • Vibecode：开发速度提升 10 倍

我的看法

这次发布改变了 Anthropic 的商业定位。从”卖模型 API”变成”卖 Agent 运行时”——沙箱、会话管理、权限控制全托管。本质上 Anthropic 开始做云服务商的事，只不过跑的不是普通应用，是 AI Agent。

架构设计上有个关键洞察：Harness 编码的假设会随模型能力提升而过时。比如 Sonnet 4.5 有”上下文焦虑”问题需要 Harness 做 context reset，但 Opus 4.5 自己就克服了。所以他们设计了一个 meta-harness——不限定具体编排逻辑，而是提供通用接口，让不同的 Harness 实现可以自由替换。

对开发者来说，最大价值是省掉了从原型到生产之间的基础设施工作。当基础设施不再是瓶颈，团队可以专注于真正重要的事：Agent 能做什么、怎么做、边界在哪里。
