前言

2026 年 4 月 4 日，GitHub Trending 呈现出几个有趣的趋势：AI 编码工具生态持续爆发，OpenAI Codex CLI 的增强层 oh-my-codex 单日暴涨 3000+ star；屏幕录制和 Demo 制作的开源替代方案获得大量关注；Google 的时序预测基础模型迎来社区热潮。让我们逐一拆解今天最热的 7 个项目。

1. oh-my-codex — 给 Codex CLI 加上「工作流大脑」

单日 +3,047 ⭐ | 总 14,263 ⭐ | TypeScript

oh-my-codex（OMX）是构建在 OpenAI Codex CLI 之上的工作流层。它不替代 Codex，而是让 Codex 变得更好用——提供标准化的工作流（从需求澄清到并行执行）、内置 Agent 团队协作、项目状态管理等能力。

为什么火？ Codex CLI 本身强大但「裸」，OMX 填补了从「能用」到「好用」的鸿沟。它的核心命令 $deep-interview（需求澄清）→ $ralplan（方案评审）→ $team（并行执行）形成了一套完整的 AI 编程工作流，这正是开发者们渴望的「最佳实践」。

技术亮点： 支持 tmux/psmux 的持久化 Agent 团队运行时，项目状态持久化在 .omx/ 目录中，还提供了与 OpenClaw 的集成指南。

适用场景： 已经在用 Codex CLI 的开发者，想要一套成熟的工作流来管理复杂编码任务。

⭐ 14,263 | 📈 +3,047/天 | 🔧 TypeScript | GitHub

2. OpenScreen — 免费的 Screen Studio 替代品

单日 +2,771 ⭐ | 总 18,301 ⭐ | TypeScript

Screen Studio 是 macOS 上知名的屏幕录制工具，月费 $29。OpenScreen 提供了一个免费开源的替代方案，覆盖了大多数人做产品演示和教程视频的核心需求。

为什么火？ 「免费替代付费工具」永远是开源社区的刚需叙事。OpenScreen 不追求 1:1 复制 Screen Studio，而是聚焦基础功能做到足够好：自动/手动缩放、多音频源录制、背景自定义、动态模糊、标注、裁剪、变速导出。

技术亮点： 基于 Electron + React + PixiJS 构建，使用 dnd-timeline 实现时间轴编辑。支持 macOS/Linux/Windows 三平台。系统音频捕获利用 Electron 的 desktopCapturer API。

注意： 项目还处于 beta 阶段，作者也坦承”可能有 bug”。如果需要专业级功能，Screen Studio 仍是更成熟的选择。

⭐ 18,301 | 📈 +2,771/天 | 🔧 TypeScript | GitHub

3. Onyx — 开源 AI 平台的「全家桶」

单日 +1,852 ⭐ | 总 23,376 ⭐ | Python

Onyx 定位为 LLM 的应用层，提供 RAG、Web 搜索、代码执行、文件生成、Deep Research 等一站式能力。支持 50+ 数据连接器和 MCP 集成，一条命令即可部署。

为什么火？ 企业级 AI 平台的需求持续旺盛。Onyx 在 Deep Research 基准测试中排名第一（截至 2026 年 2 月），这给了它强有力的技术背书。同时提供 Lite（<1GB 内存）和 Standard 两种部署模式，降低了尝试门槛。

技术亮点： 混合索引（向量+关键词）的 Agentic RAG、支持所有主流 LLM 提供商（本地和云端）、SSO/RBAC/SCIM 等企业级安全特性。

适用场景： 需要在企业内部署私有 AI 平台、整合多数据源进行知识管理的团队。

⭐ 23,376 | 📈 +1,852/天 | 🔧 Python | GitHub

4. Sherlock — 社交媒体账号搜索利器

单日 +1,192 ⭐ | 总 78,590 ⭐ | Python

老牌项目了。Sherlock 能在 400+ 社交网络上通过用户名搜索账号，是 OSINT（开源情报）领域的标杆工具。

为什么火？ 作为一个已有 78K+ star 的成熟项目，Sherlock 时不时会因为安全/隐私话题的热度重回 Trending。工具本身简单直接：输入用户名，输出该用户名在各平台的存在情况。

技术亮点： 支持 Tor 代理、多格式输出（txt/csv/xlsx）、支持通配符搜索相似用户名。通过 pipx/Docker 一键安装。

适用场景： 安全研究人员、数字取证、品牌保护（检查用户名被抢注情况）。请合法合规使用。

⭐ 78,590 | 📈 +1,192/天 | 🔧 Python | GitHub

5. TimesFM — Google 的时序预测基础模型

单日 +916 ⭐ | 总 14,172 ⭐ | Python

Google Research 出品的时序预测基础模型，最新版本 2.5 将参数量从 500M 压缩到 200M，同时将上下文长度从 2048 扩展到 16K，并支持连续分位数预测。

为什么火？ 时序预测是金融、运维、IoT 等领域的核心需求。TimesFM 2.5 的「更小更强」策略非常吸引人——参数减少 60%，能力反而增强。已被集成到 Google BigQuery 中，说明了其生产级质量。ICML 2024 论文加持。

技术亮点： Decoder-only 架构、支持 PyTorch 和 Flax 后端、去掉了频率指示器（更通用）、支持 XReg 协变量、最近还新增了 AGENTS.md 支持。

适用场景： 需要高质量时序预测的场景——金融量化、需求预测、异常检测、容量规划等。

⭐ 14,172 | 📈 +916/天 | 🔧 Python | GitHub

6. FFF.nvim — AI Agent 和 Neovim 的极速文件搜索

单日 +750 ⭐ | 总 3,289 ⭐ | Rust

FFF（Freakin Fast Fuzzy File Finder）是一个专注于文件搜索的工具，同时服务 AI Agent（通过 MCP）和 Neovim 用户。核心卖点：快，而且带记忆——基于频率、git 状态、文件大小等因素智能排序搜索结果。

为什么火？ AI 编码 Agent 的一个痛点是文件搜索效率低、token 浪费多。FFF 通过内置记忆和智能排序，显著减少 Agent 的搜索轮次和 token 消耗。对 Neovim 用户则提供了超强的模糊搜索体验（在 Linux 内核 100K 文件级别仍流畅）。

技术亮点： Rust 实现保证性能、MCP 协议集成 Claude Code/Codex 等 Agent、支持 fuzzy/grep/glob/multigrep 多种搜索模式、打字容错能力极强。

适用场景： Neovim 重度用户、使用 AI 编码 Agent 希望降低 token 消耗的开发者。

⭐ 3,289 | 📈 +750/天 | 🔧 Rust | GitHub

7. prompts.chat — 全球最大的开源 Prompt 库

单日 +375 ⭐ | 总 157,216 ⭐ | HTML

前身是「Awesome ChatGPT Prompts」，现在已进化为一个完整的 Prompt 平台。被 Forbes 报道、Harvard/Columbia 引用、学术论文引用 40+，是 Hugging Face 上最受欢迎的数据集之一。

为什么火？ 157K+ star 的老牌项目，几乎是 Prompt 工程的「起源地」。最近推出了自托管能力和交互式 Prompt 工程教程（25+ 章节），让它从一个 awesome list 进化为了一个学习平台。

适用场景： 学习 Prompt 工程、寻找灵感、为团队搭建内部 Prompt 库。

⭐ 157,216 | 📈 +375/天 | 🔧 HTML | GitHub

总结

今天的 Trending 可以提炼出三个关键趋势：

  1. AI 编码工具链持续爆发：oh-my-codex 和 FFF.nvim 分别从工作流和文件搜索切入，优化 AI 编程体验。工具不再是「能不能用」的问题，而是「怎么用得更好」。

  2. 「免费替代」叙事永不过时：OpenScreen 证明了只要做好核心功能，开源替代商业软件永远有市场。

  3. 基础模型走向垂直领域：TimesFM 代表了 Foundation Model 从通用语言向时序预测等专业领域渗透的趋势，而且「更小更强」正在成为新范式。

明天见！🚀
