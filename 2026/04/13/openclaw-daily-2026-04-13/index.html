今日要点

OpenClaw v2026.4.10 于 4 月 12 日正式发布，这是近期最重磅的一次更新。核心亮点：Active Memory 插件彻底重构了 Agent 记忆系统，Codex 原生集成、macOS 本地语音模式、以及大量稳定性修复。

🧠 Active Memory 插件——Agent 记忆的范式转变

v2026.4.10 最值得关注的功能是全新的 Active Memory 插件。此前 OpenClaw 的记忆基本是”平面文本文件”模式——MEMORY.md 加上每日笔记。现在，记忆系统升级为一个完整的知识管理体系：

  • Memory Wiki：Agent 的日常笔记、会话记录可以自动被”消化”（Dreaming/REM），生成结构化的 Wiki 页面
  • ChatGPT 导入：支持导入 ChatGPT 对话历史，让 Dreaming 系统分析并转化为可查的记忆
  • Diary View（日记视图）：Control UI 中新增时间轴导航，可以直观浏览 Agent 的记忆变迁
  • 矛盾检测：记忆系统能自动检测前后矛盾的信息，提升记忆准确度
  • 引用溯源：每条记忆都能追溯到原始来源

实际意义：以前你的 Agent “记住”事情靠的是手动写文件；现在它有了真正的”长期记忆架构”，能自动整理、关联、检索。对于长期使用 OpenClaw 的用户，这是质的飞跃。

📖 相关 PR：#64505

🤖 Codex 原生集成

OpenAI 的 Codex 编码 Agent 现在有了 OpenClaw 的原生 provider 和插件支持（Codex App-Server Harness Plugin）。这意味着：

  • 不再需要手动配置 OAuth 跳转
  • Codex 会话可以直接通过 ACP 协议管理
  • 修复了此前 Codex OAuth 登录时 invalid_scope 的问题（#64713）
  • 内部协调信息不再泄露到用户可见回复中

🎙️ macOS 本地语音 Talk Mode

Talk Mode 在 macOS 上支持本地 MLX 推理了！关键改进：

  • 首次授权麦克风后不再需要二次开关（#62459）
  • TTS 音频回复正确持久化到聊天历史中（#63514）

这让语音交互体验更加顺畅，尤其适合”解放双手”场景。

📱 Microsoft Teams 全面增强

对于企业用户，Teams 集成迎来大升级：

  • 支持 Emoji 反应（发送和读取）
  • 消息置顶/取消置顶
  • 标记已读
  • Graph API 分页优化
  • 支持委派 OAuth 设置

Agent 在 Teams 中终于像个”真正的团队成员”了。（#51646）

🔌 其他重要更新

飞书增强

飞书文档评论会话体验大幅改善——上下文解析更丰富、支持评论表情回应、打字反馈，文档评论对话更像即时聊天。（#63785）

视频生成工具升级

video_generate 工具新增 URL-only 资源交付、Reference Audio 输入、自适应宽高比等功能，为 Seedance 2.0 等下一代视频模型铺路。（#61987）

插件系统

插件清单可以声明激活和配置描述符（Setup Descriptors），减少核心代码中的硬编码特例。（#64780）

Ollama 缓存优化

模型发现阶段缓存 /api/show 返回的上下文窗口和能力元数据，反复刷新不再重复请求。（#64753）

Control UI

助手回复中的媒体、语音、引用现在渲染为结构化聊天气泡，新增 [embed ...] 富输出标签。（#64104）

🔧 重要修复汇总

问题 修复
音频转录在 OpenAI/Groq/Mistral 上失败 仅对多部分请求禁用 Pinned DNS（#64766）
WhatsApp 默认账户注册错误 使用配置的默认账户 ID（#53918）
ACP 子 Agent 进度泄漏到父会话 抑制 commentary-phase 中继文本
Google Veo 视频生成失败 不再发送不支持的 numberOfVideos 字段（#64723）
Telegram Topic 会话文件交替 统一到 canonical topic 路径（#64869）
跨 Provider 故障转移继承旧错误 限定 fallback 分类到当前尝试（#62907）
WhatsApp 图片编辑后路径丢失 保留真实媒体路径（#64918）

🛡️ 安全加固

  • SSRF 防护增强
  • Launchd 启动修复
  • WebSocket 升级连接限制，防止预认证阶段洪水攻击

升级指南

注意事项：

  • Active Memory 插件需要在 openclaw configure 中手动启用
  • 如果你使用 Codex，建议重新运行 openclaw configure 以使用新的原生 provider
  • Teams 用户需要更新 OAuth 配置以启用反应功能
