今日要点

OpenClaw v2026.4.9 于 4 月 9 日发布，核心主题是记忆基础设施化——REM 回填、日记提交/重置、持久事实提取，让 Agent 终于可以从历史日志中”学习”，而不是每次醒来都从零开始。同时大量安全加固落地，浏览器 SSRF 防护、远程节点输出消毒、不受信 .env 封锁一个不少。

🔥 版本更新：v2026.4.9

发布时间： 2026-04-09
来源： GitHub Releases

新功能亮点

1. Memory/Dreaming：REM 回填通道

这是本版最重磅的变化。新增 rem-harness --path 历史回填功能，配合日记提交/重置流程和持久事实提取，旧的 daily notes 可以重放进 Dreams 和持久记忆，不再需要维护第二套记忆栈。

简单说：你的 Agent 终于能像人一样——翻翻以前的笔记，把重要的东西”记住”。

2. Control UI：结构化日记视图

新增时间线导航、回填/重置控件、可追溯的 dreaming 摘要，以及带推广提示的 Scene 通道。运营者可以检查记忆是如何被提升的，而不是盲目信任黑盒。（PR #63395）

3. QA/Lab：Character-Vibes 评估报告

支持模型选择和并行运行，让 QA 可以更快对比候选 Agent 行为。这对做角色定制的玩家是个好消息。

4. Plugins：Provider Auth Aliases

Provider manifest 现在可以声明 providerAuthAliases，让 provider 变体共享 env 变量和认证配置，不用再给每个变体单独接线。

5. iOS：CalVer 版本锁定

TestFlight 迭代现在锁定在 apps/ios/version.json 的显式 CalVer 上，新增 pnpm ios:version:pin -- --from-gateway 发版工作流。（PR #63001）

安全修复（重点！）

修复项 说明 PR
浏览器 SSRF 防护 交互驱动的导航（点击、evaluate、hook 触发）后重新检查被禁目的地 #63226
dotenv 安全 不受信 workspace .env 文件无法注入运行时控制变量和浏览器覆盖 #62660
远程节点输出消毒 远程节点 exec 事件标记为不受信系统事件，防止注入可信 System 内容 #62659
插件 onboarding 不受信 workspace 插件无法与内建 provider auth-choice ID 碰撞 #62368
依赖审计 basic-ftp 升级到 5.2.1 修复 CRLF 注入，Hono 生产依赖 bump —

平台修复精选

  • Android 配对：新 QR 扫描时清除旧认证，后台暂停时停止自动重试（#63199）
  • Matrix：等待 sync 就绪再标记启动成功，fatal sync 停止走通道级重启而不是崩全局（#62779）
  • Slack 媒体：修复 url_private_download 图片附件因跨域 redirect 丢失 bearer auth 的问题（#62960）
  • 会话路由：保持已建立的外部路由，sessions_send 不再抢走 Telegram/Discord 的投递（#58013）
  • Gateway/Chat：内部 Agent 间控制 token（ANNOUNCE_SKIP/REPLY_SKIP）不再泄漏到用户界面（#51739）

📈 近期版本回顾

如果你还停留在旧版本，这里是近几个版本的关键变化：

v2026.4.7：推理中枢 + 记忆维基

  • openclaw infer 命令：一站式推理入口，整合模型、媒体、Web、嵌入推理
  • Memory Wiki：bundled 记忆维基栈回归，支持 sync、query、apply、结构化声明/证据字段、矛盾聚类、新鲜度加权搜索
  • 会话分支/恢复：持久化压缩检查点 + session branch/restore，压缩前状态可恢复

v2026.4.5：Dreaming 记忆系统首发

  • 实验性 Dreaming：后台记忆整合分三阶段（回忆、整理、固化）
  • 视频/音乐生成：内建 video_generate 和 music_generate 工具
  • Claude CLI 集成：通过 loopback MCP 桥向后台 Claude CLI 暴露 OpenClaw 工具

💡 实战技巧：用好 Dreaming 记忆系统

v2026.4.9 的 REM 回填功能意味着你可以把以前积累的 daily notes 全部”喂”给记忆系统。操作方式：

建议：

  1. 先用 --dry-run 预览哪些事实会被提取
  2. 在 Control UI 的新日记视图里检查推广结果
  3. 用 diary reset 清除不满意的回填批次

这比之前手动维护 MEMORY.md 高效得多——让系统自己从历史中学习。

📊 趋势观察

openclaw.com.au 的分析说得好：2026 年 AI Agent 平台的护城河不是原始自主性，而是可信赖的状态管理。

OpenClaw 最近几个版本的方向非常一致——可恢复的记忆、可检查的推理产物、更严格的工具边界、对不受信输入的显式处理。这不是在追求”做更多”，而是在追求”记得更好、解释更多、被骗更少”。
