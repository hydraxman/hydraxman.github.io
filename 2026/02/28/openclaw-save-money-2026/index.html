一个配置不当的 OpenClaw 自动化任务，一天能烧掉 1400 元 API 费。7×24 小时全量使用 Claude，月成本在 6000 到 10000 元之间。

但也有用户把月成本控制在 500 元以下，同时跑着日常助理、代码编写、内容创作等十几个自动化任务。

差距在哪？这篇文章把省钱的方案拆解清楚。

一、为什么 OpenClaw 这么烧钱？

在传统的 ChatGPT 对话模式下，一轮问答消耗几百到几千 Token。但 OpenClaw 是 Agent 模式——机器在后台持续执行任务，Token 消耗是”流量级”的：

三大烧钱场景：

  1. 多轮自我修正：一次编程任务可能经历”写代码→运行→报错→改→再跑”几十个回合
  2. 上下文无限膨胀：为让 Agent “记住”操作历史，每次调用都携带完整对话历史，活跃会话上下文轻松超过 23 万 Token
  3. 工具链级联：一个”整理邮件并创建待办”的任务，可能触发 5-10 次 API 调用，每次都带着完整上下文

36 氪引用的数据显示：OpenRouter 平台 2 月单周处理 13 万亿 Token，编程任务占比从 11% 飙到 50% 以上。Agent 驱动的输出 Token 已超过平台总输出的一半。

四、省钱方案二：技术优化

策略 1：技能固化逻辑，减少对话轮次

OpenClaw 的 Skill 系统是最大的省钱利器。把重复性工作固化成技能脚本：

原来的方式： 每一步都让 AI 思考 → 5-10 轮对话 → 消耗 50K+ Token

固化后： AI 只需一次调用触发脚本 → 消耗 2-3K Token

举个例子：微信公众号发布，从”读 Markdown → 渲染 HTML → 上传图片 → 创建草稿”全流程，AI 只需要说一句”发布这篇文章”，剩下的都是脚本自动完成。

省钱幅度：80-95%

策略 2：用 YAML 替代 JSON 做数据交换

YAML 比 JSON 少大约 30-40% 的字符数（没有引号、大括号、逗号）。在 Agent 频繁读写配置文件时，这意味着显著的 Token 节省。

策略 3：子 Agent 隔离上下文

传统 Agent 交互是线性的：每一步都带着完整历史往前走，上下文越来越大。

优化方案： 利用子 Agent（sessions_spawn）隔离任务。每个子 Agent 独立运行，完成后销毁，不污染主会话上下文。

一个 20 万 Token 的主会话拆成 5 个子任务，每个只需 3 万 Token 上下文 → 总消耗从 100 万降到 15 万。

策略 4：善用 Prompt 缓存

各家都支持 Prompt 缓存——重复使用的 System Prompt 在首次调用后大幅折扣：

  • DeepSeek：缓存命中 ¥0.2（正常 ¥2），省 90%
  • Kimi K2.5：缓存 ¥0.7（正常 ¥4），省 82%
  • MiniMax M2.5：缓存 ¥0.21（正常 ¥2.1），省 90%

实操建议： OpenClaw 的 SOUL.md、AGENTS.md 等文件作为 System Prompt 注入，保持稳定有助于缓存命中。

策略 5：批量处理非实时任务

阿里云百炼 Batch 调用直接 半价。适合：文档翻译、内容审核、数据标注、日报生成。

写在最后

Agent 时代，Token 消耗从”按次”变成了”按流量”。省钱不是抠门，是让 AI 助手能可持续地运行下去。

三句话总结：

  1. 80% 的请求不需要旗舰模型——用国产模型把大头成本砍掉
  2. 固化逻辑到脚本——AI 该做 AI 的事，脚本该做脚本的事
  3. 善用 Coding Plan + 缓存——¥200 包月 + 缓存省 90%，这是目前最低成本的组合

Token 不是金子，但也别当水泼。省下来的钱，可以让你的 AI 助手跑得更久、做得更多。
