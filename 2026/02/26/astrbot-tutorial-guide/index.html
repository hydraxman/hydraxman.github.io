AstrBot 是什么？为什么值得关注？

最近 OpenClaw 大火，但很多人不知道国内还有一个同类项目也在疯狂成长——AstrBot。

AstrBot 是一个完全免费、开源的一站式 Agentic 聊天机器人框架。简单来说：它能让你在 QQ、微信、飞书、钉钉、Telegram 等几乎所有聊天软件上部署 AI 助手。

它和 OpenClaw 的核心区别在于：

对比项 AstrBot OpenClaw
定位 多平台聊天机器人框架 个人 AI 管家
语言 Python Node.js
平台支持 QQ（个人号）、微信、企业微信、飞书、钉钉、Telegram、Discord、Slack、LINE 等 Telegram、飞书、WhatsApp、Discord、iMessage 等
QQ 个人号 ✅ 通过 NapCat/OneBot 支持 ❌ 不直接支持
微信个人号 ✅ 通过 Gewechat/Vchat 支持 ❌ 不直接支持
插件生态 1000+ 社区插件，一键安装 技能系统，手动配置
Agent 能力 MCP、Skills、沙箱代码执行 子 Agent 编排、Cron 定时任务、Memory 系统
可视化管理 ✅ 完整 WebUI 管理面板 ❌ 命令行为主
部署方式 Docker / 桌面应用 / 宝塔面板 / 1Panel / 源码 npm 安装 + 命令行配置
适合谁 想在 QQ/微信群里部署 AI 机器人的用户 想要全能私人 AI 助手的深度用户

一句话总结：想在 QQ 群/微信群部署 AI 机器人 → 选 AstrBot；想要一个能读邮件、管日程、执行自动化任务的私人 AI 管家 → 选 OpenClaw。两者也可以互补。

AstrBot 能做什么？

🤖 AI 智能对话

接入 OpenAI GPT、DeepSeek、Gemini、智谱、Moonshot、Ollama（本地部署）等几乎所有主流大模型。支持多模态（发图片让 AI 分析）、对话上下文记忆、自动压缩长对话。

🧠 Agent 智能体

  • MCP 协议支持：连接外部工具和数据源
  • Skills 技能系统：让 AI 具备特定领域的能力
  • Agent Sandbox 沙箱：在隔离环境中安全执行代码、调用 Shell
  • 网页搜索：AI 可以实时搜索互联网

💬 多平台覆盖

一套 AstrBot，同时接入多个平台：QQ 群、微信群、飞书群、钉钉群……消息统一管理，配置独立隔离。

🧩 1000+ 社区插件

插件市场里已有上千个插件，覆盖：

  • 角色扮演 / 人设定制
  • 图片生成（Midjourney、DALL-E）
  • 语音合成（TTS）
  • 知识库问答（RAG）
  • 群管理（自动审核、定时推送）
  • 游戏（猜谜、抽卡、签到）
  • 天气、新闻、翻译……

一键安装，即装即用。

💙 角色扮演 & 情感陪伴

人格设定功能，可以让 AI 扮演任何角色——猫娘、助教、客服、导游……社区有大量现成的人设模板。

首次配置：3 步让 AI 跑起来

部署完成后，打开浏览器访问 http://你的IP:6185。

第 1 步：登录管理面板

默认账号密码都是 astrbot，首次登录后会要求修改密码。

第 2 步：配置大模型

点击左侧菜单 模型提供商 → 新增。

方案 A：用 DeepSeek（国内首选，便宜好用）

  1. 去 DeepSeek 开放平台 注册
  2. 创建 API Key
  3. 在 AstrBot 中：
    • 提供商类型选 OpenAI 兼容
    • API Base URL：https://api.deepseek.com/v1
    • API Key：填你的 Key
    • 点击 获取模型列表，选择 deepseek-chat 或 deepseek-reasoner
  4. 保存并启用

方案 B：用硅基流动 SiliconFlow（注册送 16 元额度）

  1. 注册 硅基流动
  2. 获取 API Key
  3. 配置方式同上，API Base URL：https://api.siliconflow.cn/v1
  4. 可以用 DeepSeek、Qwen、GLM 等多种模型

方案 C：本地部署 Ollama（完全免费，隐私性最好）

AstrBot 中配置：

  • 提供商类型选 Ollama
  • URL：http://localhost:11434（Docker 部署的话用宿主机 IP）

方案 D：用 OpenAI / Anthropic / Gemini

直接填对应的 API Key 即可，AstrBot 原生支持这些提供商。

第 3 步：配置消息平台

点击左侧菜单 机器人 → 创建机器人，选择你想接入的平台。

接入 QQ（个人号）详细教程

这是最多人问的，详细讲一下。

原理

QQ 个人号不像 Telegram 那样有官方 Bot API。AstrBot 通过 NapCatQQ（OneBot v11 协议实现）来接入 QQ。原理是模拟 QQ 客户端登录。

步骤

1. 如果用的是 astrbot.yml 一键部署

前面已经一步到位了，NapCat 已经在运行。访问 http://你的IP:6099 打开 NapCat WebUI，扫码登录你的机器人 QQ 号即可。

2. 如果单独部署 NapCat

3. 在 AstrBot 中配置

  • 机器人 → 创建机器人 → 选择 aiocqhttp
  • 反向 WebSocket 主机地址：0.0.0.0
  • 反向 WebSocket 端口：6199
  • 在 NapCat 端也配置反向 WS 连接到 AstrBot 的地址

4. 配置管理员

在 AstrBot 的 其他配置 中，找到 管理员 ID，填写你自己的 QQ 号（不是机器人的 QQ 号）。这样你就能通过私聊机器人来管理它。

保存后 AstrBot 会自动重启并应用配置。

⚠️ 注意事项

  • 建议用小号作为机器人，不要用你的主 QQ 号
  • QQ 有风控机制，新注册的号容易被封，建议用有一定活跃度的老号
  • 不要在太多群里同时开启自动回复，容易触发风控

接入微信个人号

AstrBot 支持通过 Gewechat 或 Vchat 接入微信个人号。

⚠️ 微信个人号接入属于灰色地带，存在封号风险，建议使用小号。

具体配置参考 AstrBot 官方文档中的微信平台部分。

接入飞书 / 钉钉 / 企业微信

这些平台有官方的机器人 API，配置相对简单：

  1. 在对应平台的开放平台创建应用/机器人
  2. 获取 App ID、App Secret 等凭证
  3. 在 AstrBot 中创建对应类型的机器人，填入凭证
  4. 配置事件回调 URL（指向 AstrBot 的地址）

常见问题 FAQ

Q：Docker 镜像拉不下来？

把 soulter/astrbot:latest 换成 m.daocloud.io/docker.io/soulter/astrbot:latest。

Q：登录 QQ 一直掉线？

检查 NapCat 日志。可能是 QQ 号被风控了，换一个号试试。

Q：AI 回复很慢？

检查你配置的模型 API 延迟。国内用户建议用 DeepSeek 或硅基流动，延迟最低。

Q：能同时接多个平台吗？

可以。在 机器人 页面创建多个实例，每个实例接入不同平台。

Q：和 OpenClaw 能一起用吗？

可以互补。AstrBot 覆盖 QQ/微信等国内平台，OpenClaw 作为私人 AI 管家处理邮件、日程、自动化任务。

Q：免费吗？

AstrBot 本身完全免费开源（AGPL-3.0）。你只需要为 AI 模型 API 付费（DeepSeek 非常便宜），或者用 Ollama 本地部署实现完全免费。

写在最后

AstrBot 和 OpenClaw 代表了 AI 助手的两个方向：

  • AstrBot：聚焦”让 AI 进入每一个聊天软件”，重点是多平台覆盖和插件生态
  • OpenClaw：聚焦”让 AI 成为你的私人管家”，重点是深度自动化和工具编排

如果你想在 QQ 群里有个能聊天、能画图、能搜索、能玩游戏的 AI 机器人，AstrBot 是目前最好的选择。5 分钟部署，开箱即用，1000+ 插件随便装。
