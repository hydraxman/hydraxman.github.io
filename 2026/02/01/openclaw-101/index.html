OpenClaw 深度解读：我的 AI 助手是怎么"记住"我的？

我和 AI 助手的第一天

今天折腾了一下午，终于把 OpenClaw 跑起来了。

说实话，一开始我是冲着”本地部署”这个卖点来的——毕竟谁想让自己的聊天记录躺在别人的服务器上呢？但真正让我惊讶的，是这玩意儿居然能记住我。

不是那种”上一条消息你说了什么”的短期记忆，而是真的能记住我叫什么、喜欢什么、上周让它帮我做过什么事情。

这让我忍不住扒了一下它的代码，想搞清楚：这个记忆系统到底是怎么实现的？

先说说 OpenClaw 是什么

简单说，OpenClaw 是一个开源的个人 AI 助手框架，可以跑在你自己的电脑上。

它支持接入各种聊天平台——微信（通过 BlueBubbles）、WhatsApp、Telegram、Discord、Slack，你在哪儿聊天，它就在哪儿陪你。底层可以用 Claude、GPT，甚至本地跑 Ollama。

但今天我不想讲这些花里胡哨的功能，我想聊聊它的记忆系统——因为这才是让它从一个”聊天机器人”变成”私人助手”的关键。

记忆的本质：Markdown 文件

打开 OpenClaw 的 workspace 目录，你会看到这样的结构：

没有复杂的数据库，没有神秘的二进制格式——就是 Markdown 文件。

这个设计太聪明了：

  • 📝 人类可读：你可以随时打开看看 AI 记住了什么
  • ✏️ 人类可编辑：觉得它记错了？直接改
  • 🔒 完全可控：文件就在你电脑上，想删就删

MEMORY.md 是长期记忆，存放那些跨越时间的重要信息。memory/ 目录下按日期存放每天的笔记，就像人类的日记本。

第一层：向量索引

核心类是 MemoryIndexManager（大概 2000 多行 TypeScript），它的工作流程是：

  1. 监听文件变化：用 chokidar 监听 memory/ 目录
  2. 切分成块：把 Markdown 文件按行切成小块（默认 256 token 一块）
  3. 生成向量：调用 embedding API（OpenAI/Gemini/本地模型）把文本变成向量
  4. 存入 SQLite：用 sqlite-vec 扩展存储向量，支持相似度搜索

架构全景

除了记忆系统，简单聊聊整体架构：

层级 职责
Gateway 网关，消息路由、会话管理
Channels 各平台适配器（WhatsApp/Telegram/Discord…）
Agents AI 代理核心，处理对话逻辑
Memory 今天重点讲的记忆系统
Tools 浏览器控制、文件操作、命令执行等

代码量大概 43 万行 TypeScript，结构还算清晰。

安全设计（顺便提一嘴）

既然是跑在本地、能执行命令的 AI，安全肯定要考虑：

  • SSRF 防护：阻止 AI 访问内网地址（10.x、192.168.x 等）
  • 命令审批：三级模式（禁止 / 白名单 / 完全开放）
  • 沙箱隔离：可以用 Docker 隔离执行环境
  • 密钥扫描：防止意外提交 API key

这些我就不展开了，有兴趣可以看 src/security/ 目录。

总结

折腾了一天，我对 OpenClaw 的印象是：

✅ 记忆系统设计精巧——Markdown 文件 + 向量索引 + 混合搜索，既实用又透明
✅ 代码质量不错——TypeScript 类型完整，逻辑清晰
✅ 真的能用——不是玩具项目，是认真在做产品
⚠️ 上手有门槛——配置项多，需要折腾

如果你也想要一个能真正记住你的 AI 助手，而且不想把隐私交给云端，OpenClaw 值得一试。
