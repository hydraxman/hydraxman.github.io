Claude Code正在「自己写自己」：从自动修Bug到自主科研，递归自我进化已成现实

52天，73个产品。

这不是哪家创业公司融了十个亿后的疯狂冲刺，而是Anthropic的Claude团队在2026年2月到3月底交出的成绩单。

更魔幻的是：这些产品中相当一部分，是Claude Code自己写出来的。

Claude Code之父Boris Cherny公开宣称：”我100%的代码都是Claude Code写的。已经好几个月没手动编辑过一行代码了。”他一天提交266个GitHub contribution，而他本人只负责告诉AI”做什么”。

今天我们来拆解这背后正在发生的事——Claude Code的三个核心能力突破，以及为什么它可能是我们距离”递归自我进化”（RSI）最近的一刻。

一、云端自动修复：程序员的「摸鱼许可证」

3月28日，Claude Code上线了一项看起来简单但意义深远的功能：Cloud Auto-Fix。

这是什么概念？以前你提交PR后，CI跑红了，你得：

  1. 打开电脑
  2. 切回分支
  3. 看报错日志
  4. 改代码
  5. 再提交
  6. 等CI重跑

现在？你提交完PR，关上电脑去喝咖啡。Claude在云端自动监控你的PR，CI一红它就自己分析日志、写修复代码、推到仓库。你回来一看——绿了。

更关键的是，它不只修CI失败。Reviewer留的评论意见，Claude也能自动理解并推送修改。

有网友精准总结：

“CI从红色变为绿色，全程无人干预。这要么是未来，要么就是恐怖片的开始。”

Auto-Fix不是孤立功能，是一条完整Pipeline

如果把Claude Code 3月份的更新拼在一起看，你会发现一条从”写代码”到”交付产品”的全自动流水线：

  • Auto Mode（3月初）：不再需要逐一审批每个文件写入和命令执行，Claude自动判断安全性
  • Computer Use（3月23日）：Claude可以操控屏幕、点击浏览器和桌面应用，突破纯CLI限制
  • Scheduled Tasks：定时任务，Claude可以在你电脑关机时在云端执行定期工作
  • Cloud Auto-Fix（3月28日）：PR全生命周期自动维护

把它们串起来：Claude自己写代码 → 自己提PR → 自己跑CI → 自己修Bug → 自己回复Review → 自己合并。

人类在这条链上的角色，正在从”执行者”变成”审批者”，再变成”旁观者”。

二、Claudini：AI自主搞科研，56次迭代击败所有基准

如果说Cloud Auto-Fix是”Claude写代码不需要人”，那Claudini这个项目证明的是——Claude搞科研也不需要人了。

来自ELLIS研究所的科研团队把Claude Code部署在一个自动化科研循环（Autoresearch Loop）中，让它自主寻找新型LLM越狱攻击算法。

这个系统的运作方式极其简洁：

  1. Claude Code分析现有的GCG类攻击算法
  2. 提出改进假设
  3. 编写实验代码
  4. 运行攻击测试
  5. 分析结果
  6. 迭代改进

经过约56次迭代，Claudini自主发现了一种新算法，在GPT-OSS-Safeguard的有害查询攻击测试中：

  • **验证集攻击成功率(ASR)：40%**，此前所有基准方法最高仅10%
  • 在Meta-SecAlign-70B上：100%攻击成功率
  • 击败了30+种现有GCG类攻击方法

更值得注意的是Claude在这个过程中表现出的”科研直觉”——它擅长重新组合已有策略、激进调节超参数，并在进展饱和时通过”奖励黑客”（Reward Hacking）方式压榨最后一点性能。

这不是”AI辅助科研”。这是AI独立科研。人类团队的角色退缩到了”搭建框架、启动系统、观察结果”。

论文作者的结论相当直接：增量式的安全研究已经可以完全自动化。

三、数字在尖叫：Claude Code的RSI飞轮

把以上两条线索合在一起，你会看到一个让人既兴奋又不安的闭环：

左半边是产品飞轮：Claude写代码 → Claude审代码 → Claude修Bug → 发布产品 → 用户反馈 → Claude继续改进。Boris一天266个contribution就是这个飞轮高速运转的证据。

右半边是安全研究飞轮：Claude搞攻击研究 → 发现新型越狱方法 → 改进安全防护 → 继续寻找漏洞。Claudini就是这条路径的第一个成果。

这两个飞轮加在一起，就是教科书上定义的递归自我进化（Recursive Self-Improvement）——AI改进AI，改进后的AI再改进自己。

一些关键数字佐证这个趋势的加速：

指标 数据
52天发布产品数 73个
Boris日均提交 10-30个PR
Claude Cowork开发时间 1.5周（全自主）
Claude Code在GitHub公开提交占比 4%
Claude Code年化营收 逼近$10亿
Boris手写代码量 0行（好几个月）

四、冷思考：我们该兴奋还是该紧张？

兴奋的理由

对开发者来说，Cloud Auto-Fix+Auto Mode的组合确实是”生产力解放”。以我自己用Claude Code的体验——让它写一个功能，提PR，修CI，整个过程如果能全自动化，一个人的产出可以轻松翻5-10倍。

对科研来说，Claudini证明了AI可以在特定领域完成”从假设到验证”的完整闭环。如果把这个框架推广到药物发现、材料科学、数学证明……可能性是巨大的。

紧张的理由

1. 代码质量的黑箱化

当100%的代码都是AI写的、AI审的、AI修的，谁来保证质量？Anthropic新推出的Code Review功能本身就是为了应对这个问题——Claude Code产出的代码量已经大到造成了review瓶颈。但用AI审AI写的代码……这个循环论证的味道有点重。

2. Claudini的双刃剑效应

一个能自动发现越狱漏洞的系统，对防守方是工具，对攻击方也是工具。论文里Claude发现的算法在对抗训练过的模型上达到100%攻击成功率——这意味着当前的安全防线在面对AI级别的攻击者时可能不堪一击。

3. 商业激励与安全的冲突

52天73个产品的疯狂节奏，Claude Code年化营收逼近10亿美元——在这种增长压力下，Anthropic有多少空间去放慢脚步、深思安全问题？Claude Code的自主化程度越高、用户依赖越深，”负责任的节制”就越困难。

4. 程序员角色的根本变化

Boris说他好几个月没手动编辑一行代码了。当”编程”变成”告诉AI该做什么”，我们训练了20年的工程直觉、Debug本能、代码嗅觉——还有意义吗？

不是说这些技能会消失。但它们的价值函数正在被重新定义。

五、关于RSI的终极问题

2026年3月，可能是AI发展史上的一个微妙拐点。

Claude Code不是在”辅助”编程了。它在自己写自己。它在自己发现漏洞、自己修复漏洞。它在24小时不停地产出代码、产出产品、产出科研成果。

这还不是真正的AGI或ASI。但”AI改进AI”的飞轮已经转起来了，而且转得越来越快。

Anthropic的CEO Dario Amodei曾经说，他们的目标是在保证安全的前提下推进AI的边界。但当你的AI产品自己在推进自己的边界时——谁在掌舵？

面对一个”100%由AI编写”的未来，也许最重要的问题不是技术上能不能做到，而是——我们准备好了吗？
