从问答机器到商业引擎：AI Agent 的五层循环演进论

2023 年，AI 还只会聊天。2026 年，它已经在写代码、跑测试、部署上线。再过几年，它会管产品、做决策、驱动商业。这不是科幻——这是一条正在发生的演进路径。

核心观点

我提出一个框架，叫做 AI 的五层循环演进（Five-Layer Loop Evolution）。核心思想是：

AI 的能力边界，本质上由它所能驱动的”循环”的大小决定。 每一层循环都包含「想（Think）→ 做（Act）→ 看（Observe）」三个阶段——这是贯穿全文的母题——但验收标准和作用范围逐层扩大：

层级 循环名称 验收标准 典型产物
L0 QA Loop 回答是否正确 一段文本
L1 Task Loop 任务是否完成 一个文件/一次操作
L2 Engineering Loop 功能是否通过测试 一个 Feature/PR
L3 Product Loop 用户是否增长 一个版本迭代
L4 Business Loop 业务是否盈利 一个商业决策

AI 正在从 L0 向 L4 攀升。每一次跃迁，都伴随着工具能力、推理能力和验证能力的量变到质变。

我用一个概念来衡量 AI 在每一层的表现——AQ（Agentic Quality，智能体质量）。AQ 不是单一指标，而是三个维度的综合：

  • 规划能力：能否把一个模糊目标分解为可执行的步骤？
  • 工具运用：能否在正确的时机调用正确的工具？
  • 自我验证：能否判断自己做得对不对，并在错误时修正？

AQ 越高，AI 能驱动的循环就越大。GPT-3.5 时代的 AQ 只够支撑 L0；今天最好的模型（Claude Opus、GPT-5.4）的 AQ 刚刚够到 L2 的门槛。通往 L3 和 L4，需要的不仅是更高的 AQ，还需要一种 AI 目前完全不具备的能力——担责。这一点，我们在第五章会深入讨论。

一、历史回顾：AI 是怎么一步步长出手脚的

1.1 QA 时代（2022.11 — 2023 年中）：纯对话，无工具

2022 年 11 月 30 日，OpenAI 发布 ChatGPT（基于 GPT-3.5）。这是一个划时代的时刻——但回头看，那时的 AI 其实极其有限：

  • 没有搜索：知识截止到训练数据，无法获取实时信息
  • 没有工具：不能执行任何外部操作
  • 上下文极短：4K token，大约 3000 字
  • 只有对话：输入文本，输出文本，仅此而已

这个阶段的 AI 本质上是一个 QA Loop——用户问，它答，答完就结束。它的”验收标准”就是回答是否让人满意。没有行动力，没有记忆，没有规划。

关键词：Prompt Engineering、Zero-shot/Few-shot、ChatGPT

1.2 RAG 时代（2023 年中）：AI 的第一个工具

2023 年中期，RAG（Retrieval-Augmented Generation，检索增强生成） 概念爆发。这是 AI 获得的 第一个真正的工具调用——它第一次可以在回答之前，先去”查资料”。

RAG 的本质是什么？是 AI 入侵世界的第一个 Function Call。在 RAG 之前，AI 只能用训练时学到的知识回答问题；有了 RAG，它可以实时检索外部知识库、数据库、文档，然后基于检索结果生成回答。

这个阶段的关键词是 AIGC（AI-Generated Content）——AI 不再只是回答问题，而是开始 生成内容。但生成的基础仍然是”别人的内容”，AI 本质上是一个更聪明的搜索引擎 + 编辑器。

关键词：RAG、AIGC、向量数据库、Embedding、LangChain

1.3 Function Call 时代（2023.06 — 2024）：AI 长出了手脚

2023 年 6 月 13 日，OpenAI 正式发布 Function Calling 功能。这是一个分水岭——AI 从”只能说”进化到”能做事”。

Function Calling 让 LLM 可以：

  • 调用外部 API（查天气、订机票、发邮件）
  • 操作数据库（增删改查）
  • 触发工作流（审批、通知、部署）

LangChain 同期推出了 Agent 框架，定义了 Tool + Agent + Chain 的编排范式。AI 从一个 QA 系统，正式升级为一个 Task 执行系统。

但这个时期的 Agent 能力还很粗糙。为什么？

  • 推理能力不足：GPT-3.5 的”智商”有限，复杂规划经常出错
  • 工具集有限：每个应用需要自己定义工具，没有标准化
  • 上下文太短：4K-8K token 装不下复杂的多步骤任务
  • 没有验证机制：做完了不知道对不对

这个阶段，AI 处于 Task Loop 的初级形态——能完成简单任务，但经常需要人类兜底。

关键词：Function Calling、LangChain、Agent、Tool Use

1.4 推理觉醒（2024 — 2025）：从 ReAct 到 Thinking

AI 的 Agent 能力受限于两个核心瓶颈：推理能力和工具标准化。2024-2025 年，这两个瓶颈同时被突破。

推理范式的进化：

  • CoT（链式思维）：让 AI “一步步想”，显著提升数学和逻辑推理能力
  • ReAct：将推理和行动交织——想一步，做一步，观察结果，再想下一步。这就是 Agent Loop 的雏形
  • ToT（思维树）：不是线性思考，而是分支探索多条路径，选最优解
  • Reflection（自我反思）：AI 学会审视自己的输出，发现错误并修正
  • Extended Thinking：Claude 3.5、o1、DeepSeek-R1 等模型引入”深度思考”模式，推理链可达数万 token

与此同时，强化学习（RL） 被引入 Agent 训练。模型不再只是通过 SFT 学会遵循指令，而是通过 RLHF 和 Agent RL 学会 规划、试错、优化策略。这让 AI 的 Agentic Quality（AQ）产生质的飞跃。

关键词：ReAct、CoT、ToT、Reflection、o1、DeepSeek-R1、Agent RL

1.5 MCP 与 Skill（2024.11 — 2025）：工具标准化革命

2024 年 11 月，Anthropic 发布 MCP（Model Context Protocol）。这是 AI 工具生态的”USB 时刻”——之前每个工具都需要定制适配，MCP 定义了一套统一的协议，让 AI 可以即插即用地连接任何工具。

MCP 带来的变化：

  • 工具范围急剧扩大：从几个定制 Function，到数百个标准化 MCP Server
  • 上下文统一：工具不只是 API，还包括上下文（资源、模板、权限）
  • 跨模型通用：不绑定特定 LLM，任何模型都能调用

在 MCP 之上，出现了一个更高阶的概念——Skill（技能）。Skill 不是单个工具，而是 一组工具 + 流程 + 知识的封装。比如”发布微信公众号文章”这个 Skill，包含了 Markdown 解析、图片上传、API 调用、格式转换等多个步骤。

1.6 Computer Use 与 Agentic Coding（2025 — 2026）：AI 坐到了电脑前

2025 年，AI 获得了一项革命性能力——Computer Use（计算机使用）。Anthropic 的 Claude、OpenAI 的 Operator，让 AI 可以直接操作浏览器、终端、文件系统，就像一个人坐在电脑前工作。

与此同时，Agentic Coding 爆发：

  • OpenAI Codex（2025.05）：云端软件工程 Agent，可并行处理多个任务
  • Claude Code（2025）：终端内的自主编程 Agent
  • Cursor / Windsurf / Cline：IDE 内的 AI 编程助手

这些工具的共同特征是：AI 不再只是”写一段代码”，而是 执行整个工程循环——读代码、理解架构、写实现、跑测试、修 Bug、提 PR。

Vibe Coding 这个词在 2025 年走红——用自然语言描述你想要什么，AI 自动完成工程实现。到 2026 年，行业已经从 Vibe Coding 进化到 Agentic Engineering——AI 不是辅助你写代码，而是 自主完成整个工程任务。

关键词：Computer Use、Codex、Claude Code、Vibe Coding、Agentic Engineering

二、五层循环模型：Think → Act → Observe

有了上述历史背景，我们可以把 AI 的演进抽象成一个统一的框架。

AI 的每一层能力，本质上都是一个 Think → Act → Observe 循环。区别在于循环的 范围（Scope） 和 验收标准（Acceptance Criteria）。

各层循环的关键差异

L0 → L1 的跃迁条件：工具能力

  • 从”只会说”到”能做事”
  • 核心技术：Function Call、RAG、MCP
  • 验收：从”回答对不对”到”任务完没完成”

L1 → L2 的跃迁条件：推理 + 验证能力

  • 从”完成一个小任务”到”完成一个完整 Feature”
  • 核心技术：Extended Thinking、Computer Use、自动化测试
  • 验收：从”任务完成”到”单元测试/集成测试/UI 测试全部通过”
  • 标志事件：Codex、Claude Code 可以自主完成 Feature 开发

L2 → L3 的跃迁条件：产品感知能力

  • 从”完成一个 Feature”到”迭代一个产品版本”
  • 核心技术：用户行为分析、A/B 测试自动化、需求优先级排序
  • 验收：从”测试通过”到”用户增长/留存提升/NPS 上升”
  • 这一层需要 AI 理解 用户是谁、痛点是什么、什么 Feature 值得做

L3 → L4 的跃迁条件：商业判断能力

  • 从”迭代产品”到”驱动商业”
  • 核心技术：市场分析、成本优化、竞争策略、资源配置
  • 验收：从”用户增长”到”营收增长/利润率提升/市场份额扩大”
  • 这一层的 AI 本质上是一个 CEO Agent

三、我们现在在哪？

截至 2026 年 3 月，AI 的主流能力处于 L1 → L2 的过渡期：

L2（Engineering Loop）的当前状态：

  • ✅ 可以自主完成 Bug Fix 和小 Feature
  • ✅ 可以读懂代码库并进行重构
  • ⚡ 可以完成中等复杂度的 Feature（但需要人类 checkpoint）
  • ❌ 还不能完全自主完成大型 Feature（多文件、跨模块、复杂交互）
  • ❌ 还不能可靠地编写和维护完整的测试套件

NxCode 在 2026 年 3 月的一篇分析中指出：”全自主的多日项目仍然需要每隔几小时进行人类检查点。” 这正是 L2 尚未完全成熟的标志。

四、未来预测：通往 L4 的路径

2026-2027：L2 成熟（Engineering Loop 自闭环）

预测： AI 将能够自主完成从需求到上线的完整工程循环，包括：

  • 理解 PRD（产品需求文档）
  • 设计技术方案
  • 编写代码
  • 编写并运行测试（UT + Integration + E2E）
  • 自动 Code Review
  • 部署上线

关键突破点： 自动化测试生成和验证。当 AI 能够可靠地为自己写的代码生成高质量测试，并以测试通过作为”完成”的标准，L2 就闭环了。

2027-2029：L3 萌芽（Product Loop 初现）

预测： AI 开始参与产品决策：

  • 分析用户反馈和行为数据，自动识别优先级最高的需求
  • 自主设计 A/B 测试方案并执行
  • 基于数据决定 Feature 的 Go/No-Go
  • 自主完成从”发现问题”到”上线解决方案”到”验证效果”的完整闭环

关键突破点： AI 需要理解”什么是好产品”——这不仅是技术问题，更是对用户心理、市场趋势、竞品动态的综合判断。

2029-2032：L3 成熟 + L4 萌芽

预测： AI 驱动的产品可以自主迭代并增长用户。同时，AI 开始参与商业层面的决策：

  • 成本结构优化
  • 定价策略
  • 市场进入策略
  • 资源分配

这个阶段，人类的角色从”做事的人”转变为”定方向的人”。

2032+：L4 与 AGI 的交汇

当 AI 能够驱动完整的 Business Loop——从市场洞察、产品规划、工程实现、用户增长到商业盈利——我们可能已经非常接近 AGI（通用人工智能） 的定义了。

这不意味着 AI 取代所有人类，但意味着 AI 可以自主运转一个商业实体。人类的角色转变为：设定愿景、定义约束、提供价值判断。

五、循环的天花板：AI 做不到的那件事

前面四章讲的是 AI 能做什么、将要做什么。但如果只讲能力不讲边界，这个框架就不完整。

AI 的五层循环演进有一个隐含的假设：循环越大，验收标准越模糊、越主观、越需要判断力。在 L0 和 L1，验收是清晰的——回答对不对、任务完没完成，有明确答案。但从 L2 往上，事情开始变了：

  • L2 的验收看起来客观（测试通过），但 写什么测试、测什么标准，本身就是主观判断
  • L3 的验收是用户增长——但 什么 Feature 值得做、什么方向值得追，是品味问题
  • L4 的验收是商业盈利——但 赌哪个市场、放弃什么机会，是战略赌注

每一次往上跃迁，循环的”Observe”阶段就越依赖一种 AI 根本不具备的能力：担责（Accountability）。

5.1 AI 不会担责

这个观点听起来简单，但往深了想，它决定了 AI 演进的根本天花板。

AI 给你一个方案，执行下去亏了 100 万——谁承担？你。AI 推荐了一个方向，走到一半发现是错的——谁买单？还是你。AI 给完答案就结束了，后果跟它没有关系。它不会因为建议错了而丢工作、名誉受损、睡不着觉。

这不只是一个责任归属的法律问题。它决定了一个更大的东西——信任。

5.2 信任只能给会付出代价的人

你为什么会信一个人？不是因为他永远正确，而是因为 他错了会付出代价。正因为你相信他有东西可以失去，他说的话才有分量。

你信一个博主的推荐，是因为他推荐砸了口碑会崩、粉丝会骂，他需要为错误买单。但 AI 呢？AI 出错了什么都不会发生。没有人会说”我再也不信这个大模型了”，你最多换一个工具，而且你不会觉得被辜负。

所以人其实不会真的 信任 AI——他们会 用 它，但不会 信 它。信任这个东西，只能给到一个有可能让你失望、但选择不让你失望的人。

这就是为什么 AI 做不了 IP，虚拟偶像可以很有趣很有才华，但观众本能地知道背后没有一个真正会为判断担责的人。你可能喜欢它，但你不会信它。

5.3 品味：AI 时代最硬的硬通货

把这个逻辑推到底：如果 AI 能做一切执行工作，那人类最不可替代的是什么？

不是创造力——AI 已经能创造了。不是情感——那太虚了。是 品味（Taste）。

品味这个词以前听起来像奢侈品，好像只有搞艺术的人才需要。但在 AI 时代，品味变成了 最硬的硬通货。因为当生产力不再稀缺，判断力就成了稀缺的东西。谁能定义什么是”好的”，谁就站在价值链的最上游。

AI 能生成 1 万个方案，但它不知道哪个是好的。这个判断不是客观的——它是一个人用全部生命经验做出的主观决定。你吃过的亏、踩过的坑、被什么事情深深感动过、对什么事情忍无可忍——这些经历沉淀下来的东西，就是你的品味。

这里有一个反直觉的事：你过去”浪费”的那些经历，现在突然变得有价值了。你走过的弯路、犯过的错、那些看起来没用的爱好和没有意义的纠结——它们不是浪费，它们是品味的原材料。一个从来没被生活毒打过的人，判断是空的；一个只会跟着标准答案走的人，品味是别人的。只有你自己活过的那些东西，才能让你看到某个方案时说”对，就是这个”或者”不对，差一点”。

这个能力 AI 没有，因为它没有活过。

5.4 回到五层循环：人机共生的真正形态

把”担责”和”品味”放回五层循环框架，我们可以画出一条更完整的演进图景：

L0-L2 是 AI 主导区——在这个范围内，AI 的 AQ 足够高，验收标准足够客观，AI 可以自闭环。这是纯粹的生产力革命。

L3-L4 是人机共生区——在这个范围内，AI 负责执行和分析，但”做什么”和”值不值得做”的判断权在人类手里。不是因为 AI 不够聪明，而是因为它 无法担责，因此 无法被信任，因此 无法做最终决策。

这意味着 AI 时代最重要的个人投资，不是学一门 AI 课程，而是 花时间认识你自己——你的判断力从何而来？你对什么有真正的直觉？你愿意为什么判断承担后果？

当 AI 把你的”工具属性”剥夺干净，剩下的就是 你。问题只是：你准备好认识那个人了吗？

七、结语：循环越大，人越重要

回顾这篇分析，核心论点可以用两句话概括：

AI 的演进，是不断扩大”想 → 做 → 看”循环范围的过程。循环越大，验收标准越主观，对判断力和担责能力的要求越高。

AI 会吞噬一切可以客观验证的执行工作，但”什么值得做”和”谁来承担后果”——这两个问题永远属于人类。

我们正站在 L1 → L2 的拐点上。Codex 和 Claude Code 已经能完成简单的 Engineering Loop，但还不能可靠地自闭环。这个阶段可能持续 1-2 年。

更大的循环——Product Loop 和 Business Loop——不仅需要 AI 在技术上取得突破，更需要我们重新思考人与 AI 的关系。AI 不会取代你，但它会剥掉你身上的”工具层”，逼你面对一个问题：剥掉工具属性之后，你是谁？

这不是威胁，这是机会。当生产力不再稀缺，品味、判断力和担当就成了最稀缺的资源。你过去活过的每一天——包括那些弯路、错误和无意义的纠结——都是这种资源的原材料。

循环在扩大，工具在进化。但最大的那个循环，永远需要一个活过、错过、学过的人来闭合。
