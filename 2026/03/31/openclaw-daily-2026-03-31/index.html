今日要点

OpenClaw 连续发布 v2026.3.24 稳定版和 v2026.3.29 预发布版，带来 xAI 搜索深度集成、插件工具审批系统、MiniMax 图像生成等重磅更新。社区教程持续爆发，Ollama 零成本部署和多 Agent 编排成热门话题。

版本更新

v2026.3.29（Beta 预发布，3月29日）

这是一个内容密集的预发布版本，核心变更：

⚠️ Breaking Changes

  • Qwen 旧版 OAuth 移除：qwen-portal-auth 集成已弃用，需迁移到 Model Studio，运行 openclaw onboard --auth-choice modelstudio-api-key 完成切换
  • 旧配置迁移清理：两个月以上的自动配置迁移已移除，极老的配置 key 现在会直接报验证错误而非自动重写

🔥 重点新功能

  1. xAI 搜索全面集成：内置 xAI 提供商迁移到 Responses API，新增 x_search 一等公民支持。配置了 Grok 的用户无需手动开关插件，搜索能力自动启用。openclaw onboard 和 openclaw configure --section web 也新增了 x_search 配置向导。（#56048）

  2. 插件工具审批系统：before_tool_call 钩子新增 requireApproval 异步方法，插件可以在执行工具前暂停并请求用户确认。支持 Telegram 按钮、Discord 交互、/approve 命令等多渠道审批。（#55339）

  3. MiniMax 图像生成：新增 image-01 模型支持，包括文生图和图编辑，可控制宽高比。同时精简模型目录，仅保留 M2.7。（#54487）

  4. ACP 当前会话绑定：Discord、BlueBubbles、iMessage 支持 /acp spawn codex --bind here，直接在当前聊天中开启 Codex 工作区，无需创建子线程。

  5. OpenAI apply_patch 默认启用：OpenAI 和 Codex 模型现在默认启用 apply_patch，沙箱权限与写权限对齐。

🔧 其他改进

  • Slack 新增 upload-file 文件上传动作
  • Matrix TTS 回复升级为原生语音气泡
  • 新增 openclaw config schema 命令输出 JSON Schema
  • 内存系统：预压缩刷新逻辑移交内存插件管理
  • Podman 容器化流程简化，支持 ~/.local/bin 启动助手

🐛 关键修复

  • Anthropic 敏感内容 stop reason 不再导致 agent 崩溃
  • Gemini 3.1 Pro/Flash/Flash-Lite 模型解析修复
  • Codex 图像分析注册修复，不再报缺少 provider
  • WhatsApp 自聊天模式无限回声循环修复

v2026.3.24（稳定版，3月24日）

同样精彩的稳定版：

  • Gateway OpenAI 兼容性：新增 /v1/models、/v1/embeddings 端点，完善 RAG 客户端兼容
  • /tools 命令升级：显示当前 agent 实际可用工具，新增紧凑视图和 Control UI 实时面板
  • Microsoft Teams SDK 迁移：支持流式回复、欢迎卡片、AI 标记、消息编辑/删除
  • 技能一键安装：内置技能（coding-agent、gh-issues、weather 等）支持依赖缺失时一键安装
  • Control UI 技能管理面板：状态筛选（全部/就绪/需配置/已禁用）、详情对话框、安装按钮
  • Discord 智能线程命名：autoThreadName: "generated" 选项，LLM 自动生成线程标题
  • CLI 容器模式：--container 参数直接在 Docker/Podman 容器中执行命令
