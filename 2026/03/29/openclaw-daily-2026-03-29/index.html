版本更新：2026.3.28-beta.1

距离 3.22 正式版刚过去一周，昨晚（3月28日）OpenClaw 就推送了新的 beta 版本。这次更新信息量不小，我们挑重点聊：

⚠️ Breaking Changes

Qwen Portal OAuth 移除 — 如果你之前用 qwen-portal-auth 方式接入通义千问，这条路被砍掉了。官方建议迁移到 Model Studio，命令很简单：

旧配置自动迁移停止 — 超过两个月的老配置键不再自动修复，直接报验证错误。用 openclaw doctor 检查一下你的配置。

🔥 重点新功能

1. xAI 原生搜索集成

bundled xAI 提供商迁移到了 Responses API，并新增了 x_search 工具。最爽的是：如果你已配置 Grok 认证和 web-search，xAI 插件会自动启用，不用手动 toggle 了。（PR #56048）

2. 插件 Hook 审批机制（requireApproval）

这是一个架构级升级。插件现在可以在 before_tool_call Hook 中暂停执行、弹出审批请求。支持多渠道审批：

  • Telegram 按钮
  • Discord 交互
  • 任意频道的 /approve 命令
  • exec 审批覆盖层

对于需要人工审核关键操作的场景（比如自动发邮件、删文件），这个功能非常实用。（PR #55339）

3. ACP 频道直接绑定

新增 current-conversation ACP 绑定，支持 Discord、BlueBubbles 和 iMessage。用法：

直接把当前聊天变成 Codex 工作区，不用创建子线程。这让 Agent 协作更加丝滑。

4. MiniMax image-01 图像生成

MiniMax 终于有了图像生成能力！支持文生图和图生图编辑，还能控制宽高比。同时老模型大清理——M2、M2.1、M2.5、VL-01 全部移除，只保留 M2.7。（PR #54487）

其他值得关注的更新

更新 说明
Gemini CLI 后端 内置 Gemini CLI 推理支持，与 Claude CLI、Codex CLI 并列
Slack 文件上传 新增 upload-file Action，支持文件名/标题/评论
Matrix TTS 语音气泡 auto-TTS 回复变成原生 Matrix voice bubble
openclaw config schema 打印 openclaw.json 的 JSON Schema，方便 IDE 校验
内存插件架构 pre-compaction memory flush 逻辑移入 memory-core 插件
Tavily 来源标识 请求头加了 X-Client-Source: openclaw

🐛 Bug 修复亮点

  • Gemini 3.1 模型解析修复 — pro/flash/flash-lite 在所有 Google 提供商别名下正确解析
  • WhatsApp 无限回声循环 — 自聊模式下 bot 回复被当成新消息重新处理，终于修了！
  • Telegram 长消息分割 — 不再在单词中间断开
  • OpenAI Codex 图片分析 — 注册问题导致的图片理解失败已修复

3.22 正式版回顾（如果你还没升级）

对于还停留在老版本的同学，3.22 是 2026 年最大的一次正式版更新，核心变化：

  • ClawHub 原生集成：openclaw skills install/search 一键安装社区技能（已有 13,000+ 技能）
  • Matrix 插件：去中心化通讯协议支持
  • Anthropic Vertex AI：通过 Google Cloud 使用 Claude
  • 内置搜索：Exa、Tavily、Firecrawl 开箱即用
  • 默认模型变更：gpt-4o → gpt-5.4

⚠️ 升级前必做：检查 .env 中是否还有 CLAWDBOT_* 或 MOLTBOT_* 变量，全部替换为 OPENCLAW_*。
