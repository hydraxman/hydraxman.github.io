Claude Code Hooks 完全指南：让 AI 编程助手按你的规矩办事

你跟 AI 说「改完代码记得跑测试」，它有时候听，有时候忘。Hooks 的出现，把「建议」变成了「强制执行」——不是靠 AI 记性好，而是靠代码保证。

一句话理解 Hooks

CLAUDE.md 是风格指南，Hooks 是 Linter。

CLAUDE.md 告诉 Claude「你应该这么做」，Claude 大概率 会听。但 Hooks 不一样——它是在 Claude Code 生命周期的关键节点自动执行的脚本，没有「大概率」，只有「每次都执行」。

维度 CLAUDE.md Hooks
类型 引导（Guidance） 强制（Enforcement）
保证 Claude 通常会遵循 100% 执行
失败模式 Claude 忘了或降低优先级 脚本报错（可见、可调试）
适用场景 「优先用函数式风格」 「禁止 rm -rf 命令」

三种 Hook 类型

Claude Code 提供三种不同类型的 Hook，适用于不同的自动化场景：

1. Command Hook（最常用）

Shell 脚本，通过 stdin 接收 JSON 输入，通过退出码控制行为：

  • Exit 0：允许操作，stdout 的 JSON 会被解析
  • Exit 2：阻止操作，stderr 作为错误信息展示给 Claude
  • 其他退出码：非阻塞错误

2. Prompt Hook（判断型）

把 Hook 输入发给 Claude 模型做单轮判断，适合模糊决策：

模型返回 {"ok": true} 允许，或 {"ok": false, "reason": "..."} 阻止。

3. Agent Hook（最强大）

启动一个子 Agent，能使用工具（Read、Grep、Glob）检查代码库后再做决定。最多 50 轮工具调用：

什么时候用哪种？

场景 推荐类型
正则匹配、文件检查、跑 Linter Command
「这像不像 Breaking Change？」 Prompt
「测试全通过了吗？」需要实际跑代码验证 Agent

  • 通知类 Hook → 全局配置（~/.claude/settings.json），每个项目都生效

最佳实践

  1. 从通知 Hook 开始——最简单、最实用、零风险
  2. 用 /hooks 命令配置——Claude Code 内置了交互式 Hook 配置菜单
  3. Command Hook 优先——确定性高、调试简单、无额外 token 消耗
  4. Agent Hook 谨慎用——会消耗 token 并增加延迟，只用在高价值验证场景
  5. 测试你的 Hook——在配置后手动触发一次，确认行为符合预期
  6. 合理使用 Matcher——空字符串匹配所有，具体工具名精确匹配，| 分隔多个

写在最后

Claude Code Hooks 的本质是把人的意图变成确定性执行。

AI 编程助手最让人不放心的地方，不是它不够聪明，而是它「有时候忘」。Hooks 解决的就是这个问题——你不再需要在 Prompt 里反复强调「记得跑测试」「别动生产环境」「改完格式化」，写一次 Hook，永远生效。

这不是限制 AI 的能力，而是给 AI 装上了护栏。就像高速公路的护栏不会让你开得更慢，但能保证你不会冲下悬崖。
